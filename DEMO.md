# Demo: "День з життя агента підтримки ShopVibe"

## Про що це демо

Це покрокова демонстрація SupportAI — платформи для AI-підтримки клієнтів. Замість показу окремих фіч, ми проходимо через робочий ранок агента підтримки ShopVibe. Кожна фіча з'являється тому, що цього вимагає сюжет.

**Тривалість:** ~25-30 хв (скорочена версія ~15 хв)

---

## Підготовка перед демо

### Технічна підготовка

```bash
# 1. Засіяти демо-дані (створює workspace "demo" із 8 інструментами, 2 flows, 3 skills, базу знань)
cd backend && python3 -m seed

# 2. Запустити обидва сервіси (backend :8000 + frontend :3000)
./run.sh
```

### Налаштування екрану

- **Ліве вікно** — адмін-панель: `http://localhost:3000`
- **Праве вікно** — сайт клієнта з віджетом: `http://localhost:8000/static/widget/demo.html`
- Рекомендовано: 50/50 split або два монітори

### Чеклист перед стартом

- [ ] `seed` виконано успішно (лог: "Demo workspace seeded!")
- [ ] Обидва сервіси запущені (`run.sh` без помилок)
- [ ] Адмін-панель відкривається, відображає Dashboard
- [ ] Віджет на demo.html відкривається кліком на кнопку чату
- [ ] `.env` файл має валідний `ANTHROPIC_API_KEY`

## АКТ 1: Початок зміни — Dashboard

**Тривалість:** ~2 хв
**URL:** `http://localhost:3000`

### Що робити

1. Відкрити головну сторінку Dashboard
2. Показати 4 картки метрик:
   - **Open Tickets** — кількість відкритих тікетів
   - **Avg Response** — середній час відповіді (хвилини)
   - **AI Resolved %** — відсоток розмов, вирішених AI без людини
   - **CSAT Score** — середня оцінка клієнтів
3. Звернути увагу на жовтий банер **"N tool execution(s) waiting for approval"** — сказати, що повернемось до цього пізніше
4. Прогортати список **Recent Conversations** — показати кольорові пріоритети та статус-бейджі (open, ai_handling, escalated, resolved)

### Що казати

> "Агент починає зміну і бачить стан системи за секунду: скільки тікетів відкрито, як працює AI, і чи є дії, що чекають на людину. Це його command center."

### Перехід

> "Подивимось, звідки приходять ці розмови. Клієнт якраз пише у віджет..."

## АКТ 2: Рефанд — Живий сценарій

**Тривалість:** ~5-7 хв
**Це ключовий акт** — найповніший сценарій, який торкається tools, approvals і escalation.

### 2.1 Клієнт відкриває віджет

**Де:** праве вікно (demo.html)

1. Клікнути кнопку чату (кружок внизу справа)
2. Показати:
   - Вітальне повідомлення від AI
   - Поле вводу
   - Брендовий колір віджету

### 2.2 Клієнт пише запит на рефанд

**Що ввести у віджет:**

```
Привіт, мене двічі списали за замовлення #ORD-1001. Хочу повернення коштів.
```

**Що відбувається:**
- AI розпізнає інтент "refund" / "повернення"
- Запускає flow для рефанду, починає збирати дані
- Typing indicator → перша відповідь AI

**Що казати:**

> "AI розпізнав намір клієнта — запит на повернення коштів. Зараз він почне збирати необхідну інформацію і перевіряти замовлення."

### 2.3 Перейти в адмін-панель — Chats

**Де:** ліве вікно → `/chats`

1. Клікнути **"Chats"** у сайдбарі
2. Показати фільтри статусів: All / Escalated / AI Handling / Resolved
3. Показати зелену точку live-з'єднання (WebSocket)
4. Відкрити нову розмову
5. У треді показати:
   - **Повідомлення клієнта** — індіго бабл, справа
   - **Відповідь AI** — біле, зліва, з іконкою бота
   - **Картку tool call** — `check_order_status` з JSON параметрами (`{"order_id": "ORD-1001"}`)

**Що казати:**

> "Агент бачить все, що AI робить, в реальному часі. Кожен виклик інструменту прозорий — бачимо який інструмент, з якими параметрами, який результат."

### 2.4 AI запитує рефанд — потрібен Approval

**Що відбувається:**
- AI викликає інструмент `refund_payment`
- Цей інструмент має `requires_approval = true` → процес зупиняється
- З'являється картка **pending tool execution** з кнопками Approve / Reject

**Що показати:**
1. Pending tool execution картку в розмові
2. Повернутись на Dashboard — банер pending approvals збільшився

**Що казати:**

> "Human-in-the-loop. AI хоче повернути реальні гроші клієнту — але не може без дозволу людини. Це ключова відмінність від повністю автоматичних систем."

### 2.5 Approve рефанд

1. Натиснути **"Approve"** на картці tool execution
2. Tool виконується → AI автоматично складає підтвердження для клієнта
3. Перемкнутись на віджет — клієнт бачить підтвердження рефанду

**Що казати:**

> "Один клік — рефанд оброблений, клієнт отримав відповідь. Агент не написав жодного слова, але контролював процес."

### 2.6 Resolve

1. Натиснути **"Resolve"** — статус розмови міняється на зелений бейдж

**Перехід:**

> "Як AI знав що робити? Які інструменти має, як вони працюють? Покажу."

## АКТ 3: Під капотом — Tools

**Тривалість:** ~2-3 хв
**URL:** `/tools`

### Що показати

1. Відкрити сторінку Tools
2. Показати **8 інструментів** з бейджами:
   - **"Approval Required"** (жовтий) — `refund_payment`, `cancel_subscription`, `change_subscription_plan`, `update_shipping_address`
   - **"Auto"** (зелений) — `lookup_customer`, `check_order_status`, `reset_password`, `unlock_account`
3. Вказати на `check_order_status` (Auto) і `refund_payment` (Approval):
   > "Ці два інструменти AI щойно використав у рефанді. Один виконується автоматично, інший — чекає на схвалення."
4. Показати HTTP метод + endpoint у монокоді (GET/POST + URL)

### Тест інструменту

1. Клікнути **"Test"** на `check_order_status`
2. Ввести параметри: `{"order_id": "ORD-1001"}`
3. Натиснути Run → показати JSON результат

**Що казати:**

> "Будь-який інструмент можна протестувати прямо з адмін-панелі, не чекаючи на клієнта."

### Створення інструменту (коротко)

1. Клікнути **"Create Tool"**
2. Показати форму: Name, Method, Endpoint, Parameters JSON, **Requires Approval** checkbox
3. Закрити без збереження

**Перехід:**

> "Рефанд був відносно простий. А якщо процес потребує структури, перевірок, розгалужень? Для цього є Flows."

## АКТ 4: Візуальний конструктор — Flows

**Тривалість:** ~3-4 хв
**URL:** `/flows`

### Список flows

1. Відкрити сторінку Flows
2. Показати картки з **trigger intent** бейджами:
   - **"Refund Request"** — тригери: `refund`, `money back`, `повернення`
   - **"Account Deletion"** — тригери: `delete account`, `close account`, `видалити акаунт`

### Flow Builder — Refund Request

1. Клікнути на "Refund Request" flow
2. Показати інтерфейс:
   - **Ліва панель** — палітра з 8 типами нод: Trigger, Question, Tool, Guardrail, Condition, Response, Escalation, Skill
   - **Центр** — візуальний граф:

```
Trigger → Ask Order ID → Ask Reason → Check Amount (guardrail) → Process Refund (tool) → Confirm Refund (response)
```

3. **Клікнути на "Check Amount" guardrail:**
   - Показати: ліміт $500
   - При перевищенні → автоматична ескалація на людину

4. **Клікнути на "Ask Order ID":**
   - Показати: змінна `order_id`, валідація

### Validation

1. Натиснути **"Validate"** → зелений банер "Flow is valid"

### Account Deletion Flow (коротко)

1. Повернутись до списку, відкрити "Account Deletion"
2. Показати **Condition** ноду з розгалуженням yes/no — різні шляхи для "так, видалити" і "ні, передумав"

**Що казати:**

> "Візуально, drag-and-drop, аудитабельно. Будь-хто в команді зрозуміє процес — не потрібно читати код."

**Перехід:**

> "Flows для структурованих процесів. А як щодо загальної експертизи AI у конкретних темах? Для цього є Skills."

## АКТ 5: AI-персони — Skills

**Тривалість:** ~3 хв
**URL:** `/skills`

### Список skills

1. Відкрити сторінку Skills
2. Показати **3 скіли** з рівнями автономії:
   - **Billing Support** — Semi-Auto (AI діє, але деякі дії потребують схвалення)
   - **Technical Troubleshooting** — Manual (AI лише радить, агент виконує)
   - **New Customer Onboarding** — Full Auto (AI вирішує самостійно)

**Що казати:**

> "Три рівні автономії. Для білінгу — напівавтомат, бо це гроші. Для техпідтримки — ручний, бо кожна ситуація унікальна. Для онбордінгу — повний автомат, бо процес стандартний."

### Деталі Billing Support

1. Клікнути на Billing Support
2. Показати:
   - **Prompt template** з плейсхолдерами: `{{customer_email}}`, `{{customer_name}}`
   - **Allowed Tools** — чекбокси інструментів, доступних цьому скілу
   - **Escalation Conditions:**
     - "disputes over $500" → keywords: `dispute`, `fraud`
     - "legal threats" → keywords: `lawyer`, `sue`

### Preview

1. Клікнути **"Preview"**
2. Написати тестове повідомлення, наприклад: `I was charged twice for my subscription`
3. Показати відповідь AI в режимі Billing скіла

**Що казати:**

> "AI перемикає персону залежно від теми розмови. Кожен скіл — своя експертиза, свої правила, свої обмеження. Білінг не може робити техпідтримку, а онбордінг не торкається платежів."

**Перехід:**

> "Скіл знає як поводитись. Але звідки AI знає вашу політику повернень, ціни, правила доставки? З бази знань."

## АКТ 7: Коли все йде не так — Escalation

**Тривалість:** ~3-4 хв

### 7.1 Тригернути ескалацію

**Де:** праве вікно (віджет)

1. Відкрити нову розмову у віджеті (оновити сторінку або закрити/відкрити чат)
2. Написати агресивне повідомлення:

```
Це неприйнятно! Мене списали тричі і ніхто не допомагає! Хочу говорити з менеджером або звертаюсь до адвоката!
```

3. AI спробує заспокоїти і допомогти, але комбінація **angry sentiment + legal threat ("адвокат")** тригерить автоматичну ескалацію

### 7.2 Побачити ескалацію в адмін-панелі

**Де:** ліве вікно → `/chats`

1. Перейти на вкладку **"Escalated"**
2. Відкрити ескальовану розмову
3. Показати **три панелі:**
   - **Ліва** — список розмов
   - **Центр** — тред повідомлень + оранжевий системний банер **"Handoff Notes"**
   - **Права панель — Escalation Details:**
     - Причина ескалації
     - Тригери: `angry_customer`, `low_confidence` (червоні бейджі)
     - Sentiment: "angry" / Confidence: "low"
     - Профіль клієнта: ім'я, email, кількість повідомлень
     - Attempted Actions — які інструменти AI пробував
     - Suggested Action — рекомендація AI, що робити далі

**Що казати:**

> "AI не просто скинув розмову на агента — він дав повний контекст: чому відбулась ескалація, що вже спробував, і що рекомендує робити далі. Агент не починає з нуля."

### 7.3 Агент відповідає

1. Написати відповідь у полі для агента, наприклад:

```
Пані Іванко, мені дуже шкода за цю ситуацію. Я особисто перевірю ваші транзакції і повідомлю вас протягом 15 хвилин. Дякую за терпіння.
```

2. Повідомлення з'являється як **teal bubble** — візуально відмінне від customer (індіго) і AI (біле)
3. У віджеті клієнт бачить відповідь агента

### 7.4 Resolve

1. Натиснути **"Resolve"** — тікет закритий

**Перехід:**

> "Вся ця активність — рефанди, ескалації, інструменти — відображається в аналітиці."

2. **Pause на approval** — зробіть драматичну паузу перед натисканням Approve. "AI хоче повернути $49.99. Дозволяємо?"
3. **Два вікна** — постійно перемикайтесь між віджетом і адмін-панеллю, щоб показати обидві перспективи.
4. **Якщо AI відповідає повільно** — заповнюйте паузу поясненням, що відбувається "під капотом".
5. **Якщо щось зламалось** — Tools і Knowledge можна показати незалежно від живого чату. Переходьте до них.
6. **Ескалація (Акт 7)** — ключові слова `адвокат`/`lawyer`/`sue` надійно тригерять ескалацію. Використовуйте їх.
